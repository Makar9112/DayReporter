# Анализатор торгов СПбМТСБ

Веб-приложение для анализа торговых заявок на нефтепродукты на СПбМТСБ:
статистика, инфографика, проверка критериев недобросовестных практик и лимит заявок на инструмент.

## Стек

- Python 3.11 (рекомендуется для Streamlit Cloud; локально подойдёт 3.9–3.12)
- Streamlit, Pandas, openpyxl, Plotly, Matplotlib

## Структура

```
DayReporter/
├── app.py                 # точка входа Streamlit
├── trade_analytics.py     # статистика и графики
├── criteria.py            # проверка критериев №1–№6
├── utils.py               # парсинг времени, базис, загрузка Excel
├── requirements.txt       # зависимости
├── runtime.txt            # версия Python для Community Cloud
├── .streamlit/config.toml # тема и лимит загрузки
├── .gitignore
└── README.md
```

## Локальный запуск

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# Linux / macOS
source .venv/bin/activate

pip install -r requirements.txt
streamlit run app.py
```

Откроется браузер (обычно http://localhost:8501).


## Использование

1. В боковой панели загрузите Excel-файл (`.xlsx`) с заявками.
2. При необходимости включите **«Использую Корзину заявок»** (по умолчанию выключен).
3. По умолчанию анализируются заявки с **11:00 до 13:00** (исключается пакет «Корзины» около 10:45).
4. Задайте лимит заявок на один инструмент (по умолчанию 250).
5. Вкладки: **Общая статистика**, **Графики**, **Проверка критериев**, **Лимиты по инструментам**.
6. Скачайте HTML-отчёт кнопкой внизу страницы.

| Колонка | Обязательная |
|---------|--------------|
| Номер заявки | да |

| Код инструмента | да |
| Направл. | да |
| Цена | да |
| Статус | да |

## Базис поставки

- код оканчивается на **J** — франко-вагон станция отправления ОТП;
- на **F** — франко-вагон станция отправления;
- иначе — по подстрокам в наименовании (`труба`, `вагон`, `ОТП`).

## Критерии

| № | Суть (кратко) | Примечание |
|---|---------------|------------|
| 1 | ≥ 3 исполненных покупок за ≤ 1 с | упрощённо, без встречных заявок |
| 2 | ≥ 7 улучшений цены покупки без сделки за ≤ 1 с | упрощённо |
| 3 | ≥ 7 покупок за ≤ 1 с | исключение при «Корзине» (факт показывается) |
| 4 | ≥ 3 покупок за ≤ 100 мс | исключение при «Корзине» (факт показывается) |
| 5 | покупки у верхней границы, объём и доля ≥ 25% | исключение при «Корзине» |
| 6 | > 500 заявок на покупку по инструменту за день | месячный порог без месячных данных не проверяется |
